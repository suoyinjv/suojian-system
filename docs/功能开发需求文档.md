> 基于小麦助教功能对比，为 SchoolCMS v2.3.1 制定的功能增强方案
> 生成时间：2026-05-12
> 参考产品：小麦助教（教育信息化全场景服务商）

## 一、需求概述

### 1.1 目标
将现有 SchoolCMS（v2.3.1，基于 ThinkPHP 3.2 + AmazeUI）从"基础教务管理系统"升级为"教育机构全场景数字化运营平台"，对标小麦助教的核心功能模块。

|------|------|------|

| 成绩管理 | ✅ 正常 | 5条数据 |
| 课程管理 | ✅ 正常 | 4条数据 |

### 模块1：招生营销系统

#### 1.1 需求描述
为教育机构提供完整的招生获客解决方案，支持多种营销场景，持续招生获客，通过数据驱动高效转化意向学员。

#### 1.2 功能清单

**A. 营销活动管理**
- 活动类型：限时优惠、拼团、秒杀、优惠券、满减、折扣
- 活动配置：名称、时间、规则、适用范围、参与条件
- 活动状态：待发布/进行中/已结束/已归档
- 数据统计：访问量、报名数、转化率、成交金额

**B. 线索/意向客户管理**
- 线索来源：网站表单、线下地推、裂变海报、电话咨询
- 线索状态：新建/跟进中/有意向/已报名/已流失
- 跟进记录：时间轴展示每次沟通内容
- 自动分配：根据规则分配给课程顾问/销售

**C. 招生数据看板**
- 每日/每周/每月新增线索趋势图
- 各渠道线索转化漏斗
- 顾问业绩排名
- 转化周期分析

| 线索列表 | /Admin/Leads/index | 线索池管理 |

### 模块2：在线课堂系统

#### 2.1 需求描述
提供专业的在线教学能力，支持1v1、小班课、大班课等多种教学模式，覆盖完整的线上教学场景。

#### 2.2 功能清单

**A. 直播课程管理**
- 课程类型：1v1辅导、小班课（≤20人）、大班课（无限制）
- 课程配置：名称、科目、教师、上课时间、时长、平台
- 教师端功能：共享屏幕、白板标注、实时问答、举手发言
- 学生端功能：进入教室、观看直播、参与互动、举手提问
- 回放管理：自动录制、切片、播放器、限时访问

**B. 备课资源管理**
- 云盘空间：按课程/教师分类
- 资源类型：文档、PPT、图片、视频、音频
- 权限控制：私有/课程共享/机构共享

**C. 课前/课后通知**
- 课前提醒：自动发送上课通知（微信模板消息/短信/邮件）
- 课后作业：布置作业+截止时间
- 课后回访：自动发送课程反馈表单

### 模块3：督学打卡系统

#### 3.1 需求描述
为教培机构提供督学营销解决方案，通过多样化打卡机制创造高频互动学习场景，支持一键分享到微信朋友圈或社群，推动机构口碑营销。

#### 3.2 功能清单

**A. 打卡活动管理**
- 打卡类型：日历打卡、闯关打卡、答题打卡
- 打卡规则：每日打卡次数、连续打卡奖励、补卡机制
- 打卡模板：预设多种打卡模板（背单词/练琴/运动等）
- 数据统计：参与人数、完成率、连续打卡榜

**B. 日历打卡**
- 每日任务发布：题目/内容+打卡要求
- 学员打卡：上传图片/文字/音频/视频
- 教师点评：批改+评分+优秀展示
- 打卡日历：日历视图展示完成状态

**C. 闯关打卡**
- 关卡设计：多个关卡，每关包含多个任务
- 闯关进度：自动计算当前关卡+进度条
- 闯关奖励：完成关卡获得积分/徽章
- 关卡解锁：前关完成才解锁后关

**D. 答题打卡**
- 题库管理：单选/多选/填空/简答
- 每日题目：自动抽取/固定题目
- 自动评分：客观题自动批改
- 积分体系：答题正确获得积分

**E. 督学分享**
- 一键生成打卡海报：分享到朋友圈/社群
- 打卡排行榜：激励学员坚持
- 成长报告：每周生成学习报告发给家长

### 模块4：家校服务系统

#### 4.1 需求描述
为教培机构提供一对一家校服务能力，实现家校实时互联，服务有温度。包括学员考勤通知、重要通知传达、家长请假操作、学员成长记录、课后点评、作业布置等功能。

#### 4.2 功能清单

**A. 考勤通知**
- 考勤记录：签到/签退/迟到/请假/缺勤
- 自动通知：学生考勤状态变化时自动推送微信/短信给家长
- 考勤统计：月度/季度考勤率报表

**B. 重要通知传达**
- 通知类型：放假通知、课程调整、活动通知、作业通知
- 发送方式：微信模板消息/短信/APP推送
- 已读回执：查看家长是否已读
- 紧急通知：优先推送+短信备份

**C. 家长请假**
- 请假申请：家长发起请假（事由+时间）
- 请假审批：教师/班主任审批
- 请假记录：历史请假查询
- 批量处理：批量批准/拒绝

**D. 成长记录**
- 成长档案：图文/视频记录学生学习成长
- 标签分类：按科目/活动/时间分类
- 相册管理：按学期/年度生成成长相册
- 分享功能：分享给家长（带水印）

**E. 课后点评**
- 点评模板：预设点评模板（优/良/需改进）
- 批量点评：一次性点评多个学生
- 家长查看：家长端可查看所有点评
- 积分影响：点评影响学生积分/排名

**F. 作业管理**
- 作业布置：文字/图片/附件/视频/音频
- 作业提醒：自动通知家长
- 作业提交：家长/学生上传作业
- 作业批改：教师批改+评分+评语
- 错题本：自动收集错题供复习

### 模块7：在线课堂接入方案

#### 7.1 需求描述
接入第三方在线课堂SDK，实现1v1、小班课、大班课的在线教学能力。

#### 7.2 接入方案

**方案A：自研（WebRTC）**
- 优点：无费用，完全可控
- 缺点：开发周期长，稳定性难保证
- 适合：技术实力强，有6个月+开发周期

**方案B：腾讯云实时音视频(TRTC)**
- 优点：稳定，文档完善，按分钟计费
- 缺点：按量付费，长期使用有成本
- 适合：需要快速上线，预算充足

**方案C：阿里云RTC**
- 优点：同腾讯云
- 缺点：同腾讯云
- 适合：已在阿里云体系内

**推荐：方案B（腾讯云TRTC）+ 混合模式**
- 小班课（≤20人）：实时互动，TRTC
- 大班课（>20人）：旁听+连麦，TRTC+直播
- 1v1：超低延迟，TRTC

#### 7.3 接入参数

```env
# 腾讯云TRTC配置
TRTC_APP_ID=你的AppId
TRTC_SECRET_KEY=你的SecretKey
TRTC_USER_SIGNATURE=用户签名生成密钥

# 直播推流配置（可选）
RTMP_PUSH_URL=rtmp://push.example.com/live/
HLS_PLAY_URL=https://play.example.com/hls/
```
